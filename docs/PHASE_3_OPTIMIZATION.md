# 第三阶段：优化循环与变异 (PHASE 3: OPTIMIZATION & MUTATION)

## 1. 优化循环 (Optimization Loop)

### 1.1 核心流程
1. **初始化**: 从故事生成多个初始设计。
2. **循环过程**:
   - 生成候选设计
   - 验证过滤
   - 模拟玩家行为
   - 评分评估
   - 选择最优设计
   - 基于最优设计生成新一轮
3. **终止条件**: 达到目标评分或达到最大迭代次数。

---

## 2. 核心模块实现策略

### 2.1 Selector (选择器)
职责：从候选设计中选择最优方案。
策略：
- 保留 Top-K 设计。
- 丢弃低质量设计。
- 扩展：支持多样性保留（避免收敛过快）。

### 2.2 Mutator (变异器)
职责：基于优秀设计生成新版本。
变异方式：
- 增加/修改证据。
- 调整风险概率。
- 增加新行动。
- 修改胜负条件。
目标：在已有基础上提升复杂度与推理深度。

---

## 3. 系统核心思想
将“游戏设计”转化为一个可搜索空间：
**生成 → 验证 → 模拟 → 评分 → 选择 → 再生成**
通过循环优化不断提升游戏质量。
